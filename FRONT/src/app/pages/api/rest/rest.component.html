REST


Conjunto de restrições arquitetônicas e não um protocolo ou padrão.

Ausência de estado.

Servidores não salvam dados do cliente entre as solicitações.

Conjunto de funções:

  • POST
  • GET
  • PUT
  • DELETE

Cliente envia solicitações.

Transfere uma representação do estado.

Cabeçalhos e parâmetros também são importantes, contêm informações importantes.

  • Metadados
  • Autorização
  • Identificador uniforme de recursos
  • Armazenamento em cache
  • Cache
  • Cookies

Vários formatos possíveis de estruturar uma requisição REST.

  • JSON
    • Leitura mais simples.
    • Analisador(parsing) mais fácil.
    • JSON suporta objetos.
    • Velocidade maior na execução e transporte de dados.


  • XML
    • A linguagem é bastante flexível para representar objetos com características complexas.
    • Representa dados de uma forma muito mais natural para seres-humanos.
    • Bancos de dados XML possuem natureza auto-descritiva.
